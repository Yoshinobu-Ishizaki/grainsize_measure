# CSV 出力ガイド

grainsize_measure は解析後に 2 種類の CSV ファイルを出力します。

| ファイル | 内容 |
|---|---|
| `*_chords.csv` | 計測されたコード（弦）1 本につき 1 行（Track A — インターセプト法） |
| `*_grains.csv` | 検出された粒子 1 つにつき 1 行（Track B — 面積法） |

---

## コード CSV（`*_chords.csv`）

各行は 1 本のコードを表します。コードとは、あるスキャンラインが 2 つの粒界を連続して横切った間の距離です。

| カラム | 型 | 単位 | 説明 |
|---|---|---|---|
| `chord_id` | 整数 | — | 1 始まりの通し番号。角度 0 のスキャンから順に番号が付けられます。 |
| `length_pixels` | 浮動小数点 | px | ピクセル単位のコード長。スキャンライン上で 2 つの粒界交差点の間にあるピクセル数です。 |
| `length_um` | 浮動小数点 | µm | µm 単位のコード長（`length_pixels / pixels_per_um`）。スケールが未設定の場合は空（`null`）。 |

### コードの計算方法

1. 二値粒界画像を `line_spacing` ピクセル間隔の水平ライン群でスキャンします。
2. 各スキャンラインで粒界ピクセル（値 255）の位置を検出します。
3. 連続する 2 つの粒界交差点の間のギャップが 1 本のコードになります。
4. 画像を `theta_start` 〜 `theta_end` の範囲で `n_theta_steps` 個の等間隔角度に回転し、各角度でステップ 1〜3 を繰り返します。
5. すべての角度から得られたコードを 1 つのリストに結合して出力します。

### UI に表示される統計量

| 統計量 | 計算式 |
|---|---|
| 平均コード長（µm） | `mean(length_um)` |
| ASTM 粒径番号 G | `G = -6.6457 × log₁₀(mean_mm) - 3.298`（ASTM E112 インターセプト法） |

## 粒子 CSV（`*_grains.csv`）

各行は、すべてのフィルタ（最小面積、端部除外、ROI）を通過した 1 つの粒子を表します。

| カラム | 型 | 単位 | 説明 |
|---|---|---|---|
| `grain_id` | 整数 | — | 流域分割（watershed）アルゴリズムが割り当てたラベル番号。フィルタで除外された粒子の番号は欠番になるため、必ずしも連番にはなりません。 |
| `area_pixels` | 整数 | px² | 粒子領域内のピクセル数（粒界ピクセルを除く粒子内部の面積）。 |
| `equivalent_diameter_pixels` | 浮動小数点 | px | 同じ面積を持つ円の直径：`2 × sqrt(area_pixels / π)`。粒子を 1 つの線形サイズ指標で比較するために使用します。 |
| `centroid_x` | 浮動小数点 | px | 粒子の面積重み付き重心の水平座標（列インデックス、左端 = 0）。 |
| `centroid_y` | 浮動小数点 | px | 粒子の面積重み付き重心の垂直座標（行インデックス、上端 = 0）。 |
| `eccentricity` | 浮動小数点 | — | 粒子形状を楕円で近似したときの細長さ。**0** = 真円、**1** = 線分。0.9 を超えると強く細長い粒子を示します。 |
| `solidity` | 浮動小数点 | — | 粒子の凸包面積に対する実際の面積比：`area_pixels / 凸包面積`。**1.0** = 完全に凸（凹みなし）。0.8 を下回ると高度に不規則または凹形の粒子を示します。 |
| `area_um2` | 浮動小数点 | µm² | 物理的な粒子面積：`area_pixels / pixels_per_um²`。スケールが未設定の場合は空（`null`）。 |
| `equivalent_diameter_um` | 浮動小数点 | µm | 物理的な等価直径：`equivalent_diameter_pixels / pixels_per_um`。スケールが未設定の場合は空（`null`）。 |

### 粒子検出の手順

1. 二値粒界画像を反転し、粒子内部を `True` にします。
2. ユークリッド距離変換を計算 — 各内部ピクセルに最近接粒界までの距離が割り当てられます。
3. 距離マップの極大点（各粒子の最も内側に位置するピーク）を流域分割のマーカーにします。
4. 流域分割アルゴリズムが各マーカーから外側へ広がり、すべてのピクセルをいずれかの粒子に割り当てたラベル画像を生成します。
5. `skimage.measure.regionprops` が各ラベル領域の形状特性を計算します。
6. `min_grain_area`、`exclude_edge_grains` / `edge_buffer`、`grain_roi` に基づいて粒子をフィルタリングします。

### 形状指標クイックリファレンス

| `eccentricity`（離心率） | 粒子形状 |
|---|---|
| 0.0 〜 0.3 | ほぼ円形 |
| 0.3 〜 0.7 | 中程度に細長い |
| 0.7 〜 0.9 | 強く細長い |
| > 0.9 | 棒状またはほぼ扁平 |

| `solidity`（充実度） | 粒子形状 |
|---|---|
| 0.95 〜 1.0 | 凸形・コンパクト |
| 0.8 〜 0.95 | やや不規則 |
| < 0.8 | 高度に凹形または分岐した形状 |

---

## `null`（欠損値）について

`length_um`、`area_um2`、`equivalent_diameter_um` の各カラムは、`pixels_per_um` が未設定（UI では「(未設定)」と表示）の場合に `null` になります。物理単位での出力が必要な場合は、エクスポート前に **自動検出** または手動入力でスケールを設定してください。
